Reprenons la description d'une url :

  • le début de l'adresse (à gauche avant le premier /), correspond au nom du serveur (la machine qui héberge le site), il est strictement équivalent à un n° IP (voir le cours du semestre précédent : IP/DNS)
  • la dernière partie correspond au nom du fichier de la ressource en question (ici une page html ou image)
  • entre les deux le chemin (nom des dossiers et sous-dossiers) pour indiquer l'endroit sur le disque dur de la machine où est stocké le fichier contenant la ressource.

Dans notre exemple :

l'url de la photo de Tim Berners-Lee est 

culturenumerique.univ-lille3.fr/activitesWeb/html/tbl.jpg

elle est donc stockée sur le serveur 

culturenumerique.univ-lille3.fr

c'est à dire le même serveur que celui de la page principale.

En revanche le logo du ministère de l'enseignement supérieur et de la recherche se trouve hébergé sur un autre serveur. Son url est :

https://www.unr-npdc.org/sites/www.unr-npdc.org/files/logoMinist%C3%A8reEnsSup-200.png

et le nom du serveur est donc :

www.unr-npdc.org

Cette page est donc un  exemple d'assemblage de ressources issues de différents serveurs.
